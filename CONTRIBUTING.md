# Contributing to the ASI Cloud Agent Ecosystem

Thank you for helping us optimize this autonomous agent network architecture! To maintain compliance, please adhere strictly to these rules:

## 🔒 Security First
- **Never commit active seed phrases, private keys, or API credentials.**
- All sensitive storage mutations must pass through the configuration boundary handlers.
- Ensure all unit tests run and pass using `make test` before submitting a Pull Request.

## 🛠️ Code Base Styling
- Follow PEP 8 guidelines for Python source code structure.
- Document any mutations to message data models (`protocols/messages.py`) clearly within your pull requests.
